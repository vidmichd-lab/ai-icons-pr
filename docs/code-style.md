# Кодстайл

## Общие правила

- язык кода: `TypeScript`
- язык документации проекта: `русский`
- язык пользовательского интерфейса: `русский`
- по возможности избегать дублирования логики
- мелкие вспомогательные функции держать рядом с местом использования
- сложную логику выносить в именованные функции, а не оставлять inline

## Frontend

- UI собирать только через `shadcn/ui` компоненты
- layout и адаптивность описывать через `Tailwind` utility-классы
- не возвращаться к отдельной самодельной UI-библиотеке
- не плодить произвольные ad-hoc кнопки, модалки, карточки и инпуты
- компонент должен быть читаемым сверху вниз: state, effects, handlers, render

## Backend

- все входные payload валидировать через `zod`
- интеграции с внешними API изолировать в явные helper-функции
- не смешивать маршрутизацию handler-а и низкоуровневые сетевые вызовы
- секреты брать только из environment

## Git

- коммиты короткие и предметные
- не смешивать инфраструктурные и продуктовые изменения без причины
- документацию обновлять вместе с изменениями поведения системы
